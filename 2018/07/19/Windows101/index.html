Windows One On One

从 macOS 转换到 Windows 生态圈的自我修养

作者 Spencer Woo 日期 2018-07-19

共 2,329 字 可能要读上 9 分钟

Hi~ o(￣▽￣)ブ

我最近将自己的全部家当从 macOS 转移到了 Windows，为了更大的屏幕，更符合人类的键盘和更多的 USB 接口。当然，这也带来了卓越的性能提升和更多的折腾机会。毕竟，好看是第一生产力。

0x01 macOS 上什么让我在折腾完全部家当之后依旧流连忘返？

我不得不承认我的 12“ MacBook 性能羸弱（Intel Core™ m5-6Y54），接口贫瘠（USB Type-C × 1 和耳机孔 × 1），但是我也不得不承认这些都不妨碍我这两年来用 macOS 的体验，我在它上面用 Final Cut Pro X 剪出了我的第一个视频，用 Android Studio 编译了我的第一个安卓项目，用 iTerm2 + oh-my-zsh 的环境与 Git 项目管理工具在 GitHub 上发布了我的第一个项目，也用 Typora 写出并用 hexo 搭建了我的第一篇博客。 两年来，我真切的体验了 macOS 的生产力。

是的，我用 m5 渲染了一个五分钟的 1080p 的视频，渲染了半个小时有余，MacBook 烫的可以煎鸡蛋 🍳。

讲道理，macOS 上最让我依赖的，作为从事软件与计算机方面学习的我来说，是一套可以媲美 Linux 的开发环境。比如包管理系统 Homebrew，原生的 ssh 与 git 等等，这些大大方便了我的学习过程。据我所知，在 WSL (Windows Subsystem for Linux) 还尚未普及的时候，Windows 上用 Cygwin 或者 Git Bash 等等做代替都很难受，这也让在 Windows 上配置可用的终端环境、开发环境都异常艰难。

除此之外，在 Windows 上我也很艰苦的寻找了 iCloud，AirDrop 等等的替代品，以及无缝衔接剪贴板同步的方式。这些补救方法虽然看似拙劣，但确实提升了我同时使用 iPhone 和 Windows 的体验，可以近似的媲美 macOS 与 iPhone 的无缝联动了。


0x02 Windows 上面我都做了哪些补救？

1. 开发环境

Reference: My WSL Setup by lloydstubber

经过近两年的开发，目前 Windows Subsystem For Linux（以下简称 WSL）已经十分完善。我从微软商店下载安装了 Ubuntu 18.04 作为 WSL 的系统，并使用 Ubuntu 下的 aptitude 包管理工具链配置了我的开发环境。安装 WSL 的详细步骤在 –> 这里。

在 WSL 中我安装了我的必备开发工具：

  • git 代码版本控制
  • ssh 远程服务器连接 🔗
  • zsh 与 oh-my-zsh Shell 环境

在 Windows 中我通过 Hyper 终端环境进行实战。

Hyper 是一个基于 Election 的终端 Terminal Emulator，是一个插件丰富的、跨平台的终端。我写过一个有关它的详细介绍，在这里 –> HOW TO | 让自己的终端漂亮得不像实力派。在 Windows 下，可以通过 Hyper 调用 WSL bash.exe，这样就不用面对万恶的小黑框了 😈。

  1. 更改 .hyper.js（打开 Hyper 之后调用快捷键 Ctrl + , 来开启）将 shell 一项改为：shell: 'C:\\Windows\\System32\\bash.exe'，重启终端，这样 Hyper 就可以开启 WSL 环境；
  2. 安装主题与插件：WSL 下直接运行 hyper 命令有问题，可以通过直接在 .hyper.js 中的 plugins 一项中添加主题或插件名称来安装；

2. 效率工具

Reference: Awesome Windows on GitHub

① Wox - An effective launcher for windows 和 Everything - The fastest file/folder search tool by name

这两个完美结合，可以媲美 macOS 上的 Alfred。

Wox 是 Windows 上大名鼎鼎的效率工具，免费、开源，我派对它有详尽的介绍 –> 在这里。

② QuickLook - 一个空格键预览工具

macOS 上的空格预览文件内容绝对是一个经典，用过的人赞不绝口。在 Windows 上，这款 QuickLook 程序可以完美实现空格预览的功能，并且它可以在 Windows 应用商店下载得到，因此不必担心安全问题。

③ 快贴 - 真正的云剪贴板

Reference: 少数派 - 好用的全平台剪切板工具，我们为你找到了这 3 款

快贴是一个跨平台的剪贴板同步工具，虽说体验肯定不如 macOS 闭源的生态系统好，但是至少我不必用微信当作我电脑与手机沟通的渠道了。

④ Send Anywhere

Reference: 少数派 - 免费全平台的文件分享利器：SendAnywhere

Send Anywhere 将文件上传到一个 p2p 网络上面，并非其服务器上，通过随机六位接收码进行文件接受。这样的传输方式保证了文件的安全性和完整性，又能有相当的上传、下载和传输速度。Send Anywhere 可以说是全平台 Airdrop 了。

同时 Send Anywhere 有设备记忆功能，于是在曾经传输过文件的设备上，下一次传输连那个六位接收码都不必输入了。这样的分享文件的特性可以说是跨平台的救命稻草了。

⑤ smpic - 开源的 sm.ms 图床上传软件

我的博客图片全部都放到了第三方的图床上面，这让我需要有一个便捷的图片上传途径。

在 macOS 上面，这篇文章里面：SM.MS - 一名优秀的图床的艺术之旅 我介绍了利用 Automator 实现一个全局上传并自动转换为 Markdown 格式的图片路径的解决方案。在 Windows 上也有同样的便捷手段。

smpic 通过 SM.MS 图床的 API 与大名鼎鼎的 AutoHotKey 脚本实现了这样的图片上传利器，

⑥ Typora - 可能是 Windows 上最好的 Markdown 编辑器

Typora 不必多说了，基本上是 Windows 上写 Markdown 的必备利器了。我的这篇文章就是在 Windows 的 Typora 上面编写完成的。我派对 Typora 也有详尽的介绍，在这里 –> 让 Markdown 写作更简单，免费极简编辑器：Typora

⑦ Snipaste - 可能是 Windows 上最好的截图工具

Snipaste 着实是 Windows 上最好的截图工具。但 Snipaste 除了我们常见的截图标注、窗口检测、全局快捷键等等简朴必备技能，还有贴图、取色等等高阶可玩性。

Snipaste 的开发者对 Snipaste 很是上心，我派单独采访了这位同学，在这里有更加具体的介绍 –> 幕后丨他做了最强免费「截图」工具 Snipaste 后，还有上万字的话想说

上面介绍的工具中，除了 ⑤ 和 ⑥ 这两个外，剩下的都有幸被我加入开机自启动名单。🎉

3. 桌面 Windows 化

① TranslucentTB - 开源的任务栏透明小工具 ⭐⭐⭐⭐⭐

Windows 10 上纯粹的任务栏透明化软件：TranslucentTB 免费、开源、绿色，完美解决了 Windows 任务栏透明的问题，它利用了 Windows 10 API，简洁小巧。五星推荐。

② Rainmeter - 大名鼎鼎的 Windows 桌面专业美化插件

对于深度美化 Windows 桌面的同学来说，Rainmeter 的重要性可想而知。我只小小利用了其桌面时钟的显示。

③ Simple Desktops - 极简风格壁纸提供站点

Windows 美化方面，壁纸可以说是重中之重。一张简单可爱的壁纸可以奠定整个工作的态度。在 Simple Desktops 里我总能找到可爱的壁纸。

④ 极简风格的审美要求和苛刻到极致的完美追求

这一方面就是仁者见仁智者见智了。简洁的文件管理、不复杂凌乱的桌面和长清理的使用习惯会让你的 Windows 使用体验蒸蒸日上。💗


0x03 体验呢？

要不是为了你这么舒服的键盘、这么大的屏幕、可以接受的轻巧、以及三个标准 USB 插口，我才不会抛弃我的 MacBook！😈

确实，通过上面的折腾，我已经能够让自己在 Windows 上的开发、学习与撰写文章的体验近似 macOS 了，同时得益于我现在这台 Windows 电脑 (ThinkPad X1 Yoga) 出色的显示性能 (2K OLED) 我并没有失去 MacBook Retina Display 的使用体验，反而性能提升让我能够更加自由方便的使用 Photoshop 和 Premier 了。

这台二合一的显示色彩着实出色，其触屏和 4096 压感的 Wacom 手写笔也让我能够充分发挥其笔记、绘画和 PDF 的标注功能。


0x04 Finally

虽然服役两年的 MacBook 因为被动散热和 m5 处理器让我无奈使其退役，但是这款 MacBook 依旧是我使用过最为美丽的电脑之一。

我看过太多人被习惯绑架。比如，用 IE 浏览器习惯了，不习惯用 Chrome；用 Windows 习惯了，觉得 macOS 用起来别扭。哪怕是买了 Mac 笔记本也要去安装一个盗版 Windows；用 iPhone 习惯了，用不了 Android；用一门编程语言用习惯了，不想学第二门编程语言。

人不能被工具禁锢啊。
